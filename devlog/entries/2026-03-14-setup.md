# Devlog #01 — Project Setup & Parallax Background

## What I worked on
Started this project originally to mess around with ImGui + SFML, 
previously i couldn't get ImGui working with SFML 3.0.0.

Objective shifted pretty fast though — watched some Terraria footage 
and got obsessed with how the background layers move, wanted that for 
my game. so parallax effect became the first real task.

## What I implemented
**Project Structure**
Got the base architecture set up. everything is hierarchical —
upper classes own and manage their children, lower classes never 
reach upward and create objects above them.

```
Engine
└── World
    ├── InputHandler
    └── ParallaxBackground
        └── Background
```

**Parallax Background**
- Multiple background layers each moving at different speeds
- Background images move backward to fake the depth effect
- Infinite scrolling so the background never runs out

## Problems I ran into
`sf::View` was being annoying — the background layers would start 
moving a few frames *after* the camera starts moving. creates this 
weird rubber-band feel where the background kinda snaps to catch up. 
technically i was getting parallax effect, but it feels very weird.

## How I solved them
Ditched `sf::View` entirely for the background. instead i move the 
background images backward manually to fake the parallax effect. 
works way better, feels smooth now.

not the cleanest solution but it works for now.

## What's next
- Figure out how to get `sf::View` working properly without the delay
