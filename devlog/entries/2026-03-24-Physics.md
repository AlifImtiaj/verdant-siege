# Devlog #04 — Physics and ImGui Implementation

## What I worked on
Today I mainly worked on ImGui implementation and also spent time planning how to approach the physics system.

## What I implemented
**Implemented ImGUI**<br/>
ImGui is a popular GUI library for C++. It is very useful for someone like me who is developing a game in C++, as it allows quick and efficient creation of development tools and interfaces.

**Structuring around Physics**<br/>
I am still in the planning phase for implementing physics, specifically for static, kinematic, and dynamic objects. Since this is my first time working with a system like this, I am focusing on understanding and structuring it properly before implementation.

## Problems I ran into
**ImGui Version Mismatch**<br/>
I initially overlooked the version requirement mentioned in the repository, which specified `Dear ImGui >= 1.91.1, < 1.92.0`. Instead, I used the latest version, which caused assertion errors in debug builds. Interestingly, the release build was working correctly, which made the issue more confusing.

## How I solved them
**ImGui Version Mismatch**<br/>
After identifying the version requirement, I tried the latest version recommended again but encountered additional issues. Eventually, I settled on ImGui version `1.91.1`, which resolved the assertion problems and ensured stable behavior in both debug and release builds.
